# 点検表機能 設計まとめ

## 1. 設計方針

-   点検表定義と点検結果を分離する。
-   病院ごと・機種ごと・型式ごと・点検種別ごとに点検表を作成可能とする。
-   点検結果はDBに保存し、PDFは必要時に生成する。
-   PDFは帳票であり、DBの正本データから生成する。

# 3. PDF設計

## 基本方針

-   PDFはDBから生成する。
-   DBのみを正本とする。
-   PDFテンプレート機能は持たない。
-   1PDF = 1点検種別

例

    使用中点検

    点検結果①
    点検結果②
    点検結果③

------------------------------------------------------------------------

## 1点検結果のレイアウト

    病棟
    病室
    ベッド
    管理番号
    機種
    型式
    点検日時
    点検者

    ----------------

    点検項目
    点検項目
    点検項目
    ・・・

これを1つのカード（ブロック）として表示する。

------------------------------------------------------------------------

## ページレイアウト

-   A4縦
-   カードを上から順番に配置
-   ページに収まるだけ配置
-   収まらなくなったら改ページ
-   1ページあたり約5件を目安（項目数に応じて可変）

# 4. 個人情報

標準仕様

-   患者名はPDF必須項目としない
-   病棟
-   病室
-   ベッド
-   点検日時
-   管理番号

を表示する。

必要に応じて病院設定で患者名を利用可能とする。

------------------------------------------------------------------------

# 5. 検索思想

検索条件は画面操作のためのもの。

PDFには検索条件は印字しない。

PDFには各点検結果が持つ情報を表示する。

これにより、

-   病棟単位
-   機器単位
-   患者単位（利用時）
-   月次点検

など様々な検索結果を同一のPDF構造で出力できる。
