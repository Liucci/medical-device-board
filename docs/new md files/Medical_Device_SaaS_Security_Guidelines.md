# 医療機器管理SaaSにおけるセキュリティ対策まとめ

## 基本方針

医療情報システムのセキュリティは、**第三者からの攻撃対策**だけではなく、**正規利用者による誤操作・内部不正・情報漏えい防止**を重視する。

厚生労働省「医療情報システムの安全管理に関するガイドライン」の考え方では、技術そのものよりも、

-   誰がアクセスできるか
-   何ができるか
-   何をしたか追跡できるか
-   障害時に復旧できるか

を実現することが重要である。

------------------------------------------------------------------------

# セキュリティ対策の分類

## 1. 第三者からの攻撃対策

### 実装すべき項目

-   HTTPS(TLS)による通信暗号化
-   SQLインジェクション対策
-   XSS対策
-   CSRF対策（必要な画面）
-   認証トークン(JWT)管理
-   WAF導入（必要に応じて）
-   DDoS対策（クラウドサービスを利用）
-   マルウェア対策
-   OS・ライブラリ・ミドルウェアのセキュリティアップデート
-   脆弱性管理

------------------------------------------------------------------------

## 2. 利用者（内部）対策【最重要】

### 認証

-   個人ごとのアカウント
-   共通アカウント禁止
-   MFA（将来的に導入）

### 認可

-   ロール管理
-   RLSによるアクセス制御
-   hospital_idによる病院間データ分離
-   最小権限の原則

### 操作制御

-   重要操作時の再認証
-   一定時間無操作で自動ログアウト
-   退職者・利用停止者の即時無効化

------------------------------------------------------------------------

# 監査ログ

重要操作は必ず記録する。

最低限記録する内容

-   操作者
-   操作日時
-   対象データ
-   操作内容
-   IPアドレス（可能なら）
-   結果（成功・失敗）

対象例

-   ログイン
-   ログアウト
-   ユーザー登録
-   権限変更
-   病院情報変更
-   機器登録
-   機器削除
-   機器移動
-   点検結果変更

監査ログは原則削除不可とする。

# SaaS事業者としての責任

SaaSでは主に以下を事業者が担当する。

-   アプリケーション保守
-   セキュリティアップデート
-   バックアップ
-   脆弱性対応
-   障害対応
-   ログ管理
-   サーバ監視

医療機関側は

-   利用者管理
-   パスワード管理
-   院内端末管理
-   職員教育

などを担当する。

# セキュリティの考え方

医療情報システムでは、

「ハッカーから守る」

だけでは十分ではない。

より重要なのは

「正規利用者による誤操作・内部不正・情報漏えいを防止し、すべての重要操作を追跡できること」

である。

したがって、本システムでは

-   第三者からの攻撃対策
-   内部統制
-   監査ログ
-   権限管理
-   責任分界

を中心とした設計を行う。


データベース構造（テーブル、RLS、トリガー、関数等）はGitHubリポジトリでバージョン管理を行う。業務データについては、Backendが定期的に全テーブルをCSV形式でエクスポートし、暗号化したバックアップファイルを運営会社が管理する専用ストレージへ保存する。バックアップは運営会社のみがアクセス可能とし、各医療機関へは提供しない。障害発生時には、GitHubのSQL定義とCSVバックアップを用いてシステムを復旧する。
